Des débuts laborieux...






Il est parfois difficile d'analyser du code, il suffit de chercher dans les fichiers de ce site :
L'écriture du code dépend malgré tout de celui qui l'écrit même si le language est le même dans la logique. Les sociétés nous donnent les moyens pour communiquer avec les ordinateurs de bien des manières.
Les différents languages ont des syntaxes particulières, ils suivent un paterne, patron ou placeholder dans lequel des gens s'amuse à créer.
La plus part des gens se sente perdu lorsqu'ils sont face à du code. Il faut reconnaitre le language, les fonctions et la logique de celui-ci.
Mais encore une fois cette tâche est difficile puisque c'est un humain qui l'écrit avec sa logique de fonctionnement qui est parfois imparfaite.
Il faudrait presque que les auteurs nous donnent les clés pour déchiffrer le language si particulier qui est unique à leur programme.

Il serait bon d'apprendre à chacun comment apprendre et se munir de ces outils cependant cela est très long pour la minorité qui le pratique aujourd'hui.
Bien que l'informatique soit un outils formidable il est difficile à appréhender et constitue donc un défi pour les générations futures. Un homme qui sait lire du code sera bientôt plus utile qu'un ouvrier.
Il est donc bon d'essayer de s'adapter et de s'instruire par soit-même si l'on apprécie cette pratique tôt pour continuer à s'enrichir et à s'amuser.
Des projets simples sont possibles, bot discord, site web personnalisés, regroupement en association pour partager ensenble les clés du progrès.
